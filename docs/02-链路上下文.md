# 链路上下文

`TraceContext` 是完整传播上下文，包含 trace id、span id、parent span id、trace flags 和 opaque `tracestate`。`SpanContext` 是 span 热路径使用的轻量身份，只保留 trace/span/parent/flags；日志使用更小的 `LogContext`。

## ID 和格式

- `TraceId` 固定 16 字节，对应 32 位 lowercase hex。
- `SpanId` 固定 8 字节，对应 16 位 lowercase hex。
- `fromHex()` 接受大小写 hex，但输出统一 lowercase。
- all-zero ID 无效，`random()` 不返回 zero。

```cpp
auto traceId = galay::tracing::TraceId::fromHex("4bf92f3577b34da6a3ce929d0e0e4736");
auto spanId = galay::tracing::SpanId::fromHex("00f067aa0ba902b7");
auto context = galay::tracing::TraceContext(traceId, spanId, 0x01);
```

## 当前上下文

```cpp
galay::tracing::setCurrentContext(context);
auto current = galay::tracing::currentContext();
galay::tracing::clearCurrentContext();
```

当前上下文存在线程本地状态里。同步代码可以依赖 `startSpan()` 和 `SpanGuard` 自动设置和恢复；跨线程、异步回调、协程恢复点不应该隐式依赖 thread-local。

## SpanGuard 作用域

```cpp
{
    auto root = galay::tracing::startSpan("root");
    {
        auto child = galay::tracing::startSpan("child");
        // child is current.
    }
    // root is restored.
}
// previous context is restored.
```

`SpanGuard` move-only，析构不抛异常。默认不记录 span start/end timestamp；需要时显式开启：

```cpp
galay::tracing::setSpanTimingPolicy(galay::tracing::SpanTimingPolicy::kEnabled);
```

默认关闭 timestamp 是为了降低 span 热路径成本。日志自身仍有日志时间戳。

## Span 语义字段

`Span` 支持 kind、status 和 bounded attributes：

```cpp
galay::tracing::Span span("GET /orders", context);
span.setKind(galay::tracing::SpanKind::kServer);
span.setStatus(galay::tracing::SpanStatusCode::kError, "timeout");
span.setAttribute("http.method", "GET");
span.setAttribute("http.status_code", 503);
```

attributes 会复制 key/value，适合进入 batch queue 后异步导出。默认最多记录 `Span::kMaxAttributes` 个 attribute，超过上限时 `setAttribute()` 返回 `false`。

## Sampling

默认 sampler 是 parent-based：如果存在有效父上下文，就继承父级 sampled flag；root span 默认采样。

```cpp
galay::tracing::AlwaysOffSampler off;
galay::tracing::setSampler(&off);

auto span = galay::tracing::startSpan("not_exported");
assert(!span.span().spanContext().sampled());

galay::tracing::setSampler(nullptr); // restore default
```

需要按 trace id 做固定比例采样时：

```cpp
galay::tracing::TraceIdRatioSampler ratio(0.1);
galay::tracing::ParentBasedSampler sampler(ratio);
galay::tracing::setSampler(&sampler);
```

`setSampler()` 不持有 sampler，调用方必须保证 sampler 生命周期覆盖配置生效期间。

## 协程和调度器

`galay-kernel` 场景下推荐显式捕获 context 并传给日志：

```cpp
auto captured = galay::tracing::captureTraceContext();
co_await someAsyncWork();
galay::tracing::log(captured).info("after await");
```

这避免了“协程恢复到另一个线程后 thread-local 不一致”的问题。当前库没有全局 task resume hook；这是刻意保持低侵入的设计。
