# 日志关联

日志关联的目标是：业务代码打日志时，如果调用点存在 trace context，日志记录立即复制 trace/span 身份。sink、`spdlog` 或异步 logger 后续在哪个线程写出都不影响关联正确性。

## 普通日志

```cpp
auto span = galay::tracing::startSpan("checkout");
GALAY_LOG_INFO("order accepted {}", orderId);
```

禁用级别会先检查 level，再执行 `std::format`，因此不会格式化被过滤的日志。

## 显式 Context

协程、线程池和回调代码推荐使用显式 context：

```cpp
std::optional<galay::tracing::TraceContext> ctx = galay::tracing::currentContext();
galay::tracing::log(ctx).info("scheduled task started");
```

也可以使用显式 writer：

```cpp
galay::tracing::Logger logger;
galay::tracing::log(ctx, logger).warn("retry {}", attempt);
```

## 结构化事件

结构化事件适合高频路径和字段化日志：

```cpp
galay::tracing::event(ctx).info(
    "order_sent",
    galay::tracing::field("order_id", orderId),
    galay::tracing::field("symbol", "IF2406"));
```

字段的 string 值是借用的，writer 必须在 `write(StructuredLogRecord)` 返回前消费字段，不要保存 `fields` span。

## 宏快路径

需要避免 disabled event 构造字段时，使用宏：

```cpp
GALAY_EVENT_INFO_DEFAULT(
    ctx,
    "order_sent",
    galay::tracing::field("order_id", orderId));
```

`GALAY_EVENT_*` 和 `GALAY_EVENT_*_DEFAULT` 会先检查 level，disabled 时不会求值字段表达式。

## 自定义 Writer

自定义 writer 不需要继承虚基类，只要满足 concept：

```cpp
class Writer {
public:
    bool isEnabled(galay::tracing::LogLevel level) const noexcept;
    void write(galay::tracing::StructuredLogRecord record);
};

Writer writer;
galay::tracing::setDefaultLogWriter(&writer);
```

默认 writer 是非拥有引用。调用方必须保证 writer 生命周期长于默认 writer 配置，或者在销毁前调用：

```cpp
galay::tracing::setDefaultLogWriter(nullptr);
```

## spdlog

`SpdlogSink` 接收已经带有 `LogContext` 的 `LogRecord`，不会依赖 `spdlog` MDC。这样即使用 `spdlog` async logger，trace/span id 仍来自业务调用点。
