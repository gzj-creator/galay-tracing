# 快速开始

`galay-tracing` 的最小使用路径是：创建 span，业务代码照常打日志，需要跨服务时解析和注入 W3C Trace Context。

## 构建

```bash
cmake -S . -B build-dev -DBUILD_TESTING=ON -DBUILD_EXAMPLES=ON -DBUILD_BENCHMARKS=OFF
cmake --build build-dev
ctest --test-dir build-dev --output-on-failure
```

## 创建本地 Span 并打日志

```cpp
#include "galay-tracing/kernel/span_guard.h"
#include "galay-tracing/log/logger.h"

void checkout() {
    auto span = galay::tracing::startSpan("checkout");
    GALAY_LOG_INFO("order accepted {}", 123);
}
```

`startSpan()` 会设置当前线程的 trace context，`SpanGuard` 析构时恢复之前的 context。日志会在调用点复制轻量 `LogContext`，不会依赖 sink 线程上的 thread-local 状态。

## 处理入站 Trace Context

```cpp
#include "galay-tracing/context/context_storage.h"
#include "galay-tracing/context/traceparent.h"
#include "galay-tracing/kernel/span_guard.h"

bool handleRequest(std::string_view traceparent, std::string_view tracestate) {
    auto inbound = galay::tracing::extractTraceparent(traceparent, tracestate);
    if (!inbound.has_value()) {
        return false;
    }

    auto span = galay::tracing::startServerSpan("GET /orders", *inbound);
    return galay::tracing::currentContext().has_value();
}
```

`startServerSpan()` 会沿用入站 trace id，生成新的 span id，并把入站 span id 记录为 parent span id。

## 注入出站 Trace Context

```cpp
auto current = galay::tracing::currentContext();
if (current.has_value()) {
    auto traceparent = galay::tracing::injectTraceparent(*current);
    auto tracestate = galay::tracing::injectTracestate(*current);
    // Put traceparent/tracestate into outbound request headers.
}
```

`tracestate` 当前按 opaque string 保存和透传，不解析 vendor 语义。

## 运行示例

```bash
cmake --build build-dev --target E1-LogContext E2-Traceparent
./build-dev/bin/E1-LogContext
./build-dev/bin/E2-Traceparent
```
